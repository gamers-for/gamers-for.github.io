# スプラトゥーン3 セクション リビルド計画

## やること（一言）

Game8のスプラ3ページをスクレイプ → データ抽出 → 文章全部書き直し → アイコンにGF透かし → 大画像はプレースホルダー → Hugo記事として公開

## ステップ

### Step 1: Game8スクレイプ（データ取得）

`scripts/scrape_game8_splatoon3.py` を作成

**取得対象:**
- 武器ティアリスト（総合 + ルール別5つ）
- 武器個別ページ（~160ページ）→ 強い点・弱い点・立ち回り・ギア推奨
- ステージ一覧 + 個別ページ（~25）
- ギアパワー詳細
- ギア一覧（頭・服・靴）
- サーモンラン（ボス攻略・ステージ・ランク）
- バトルモードガイド（5ルール）
- 初心者ガイド群

**保存先:** `００００１スプラトゥーン３/raw_html/` にHTML + 抽出テキスト

**ルール:** 1-2秒間隔でリクエスト、合計~210ページ

### Step 2: データ解析・統合

`scripts/parse_game8_splatoon3.py` を作成

- HTMLからデータ抽出 → JSON化
- 既存の `splatoon3_master.json` とマージ
- 武器ごとに追加: 強み・弱み・立ち回りコツ・ルール別評価・ギア推奨・相性情報
- ステージ詳細・ギア一覧・サーモンラン詳細を追加

**出力:** `００００１スプラトゥーン３/parsed_data/` → 検証後 `data/splatoon3_master.json` に統合

### Step 3: アイコン処理

`scripts/watermark_icons.py` を作成

- 既存の459枚武器アイコン + サブ + スペシャルアイコン
- 全てに「GF」透かしを左下に追加（半透明、小さく）
- WebP形式で保存（quality 85）
- 元PNGはバックアップとして残す

### Step 4: プレースホルダー画像生成

`scripts/generate_placeholder.py` を作成

- ステージ画像（25枚）: 800x450、「画像準備中」テキスト
- ギアアイテム画像: カテゴリ別
- サーモンランボス画像
- サイトカラーで統一感あるデザイン

### Step 5: コンテンツ生成（メイン作業）

`scripts/generate_all_pages.py` を大幅強化

**文章の口調:**
- カジュアルで楽しい感じ
- 「〜だよ！」「マジで強い」「ぶっちゃけ」等
- 実用的なアドバイス重視
- 情報源は一切書かない（自分たちの分析として書く）

## ディレクトリ使い分け

| ディレクトリ | 用途 |
|---|---|
| `０００００テンプレ/` | テンプレート・計画書・設計書 |
| `００００１スプラトゥーン３/` | 実際の作業場（スクレイプ・パース・試行） |
| `content/games/splatoon3/` | 最終公開先（検証後に配置） |
| `static/images/games/splatoon3/` | 画像最終配置先 |

**作業フロー:**
1. テンプレをまず `０００００テンプレ/` に書く
2. `００００１スプラトゥーン３/` で試す
3. OKなら `content/` と `static/` に配置

---

## 禁止事項（絶対守る）

1. Game8, GameWith, Altema の名前をコンテンツに**一切書かない**
2. テーブルセル内に画像を入れない
3. テンプレ文章で埋めない（Game8から取った実データを使う）
4. HTMLコメント内でも禁止ワードNG
