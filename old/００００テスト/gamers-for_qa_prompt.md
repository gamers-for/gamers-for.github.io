# Gamers-For 攻略サイト制作・品質チェックプロンプト

## 概要

本ドキュメントは、Gamers-For攻略サイトの各ページ制作時および制作後の品質チェックに使用するプロンプト集である。Claude Code等のAIエージェントに投げて自動チェックさせることを想定。

---

## A. ページ制作時の指示プロンプト

### A-1. 情報収集・統合ルール

```
【絶対禁止事項】
- 他サイト名（Game8、GameWith、アルテマ等）への言及を一切含めないこと
- 「参考にした」「引用した」「参照した」等の文言を絶対に使わないこと
- 出典表記、参考リンク、引用元の記載を行わないこと

【情報の網羅性チェック】
以下を確認すること：
1. 収集した複数ソースの情報の「和集合」になっているか
   - 項目数：全ソースに登場する全項目を網羅しているか（A にあって B にない項目も含める）
   - 項目の中身：各項目の説明文は、全ソースの情報を統合・要約したものか
   - 情報が薄い箇所がないか：特に以下を重点チェック
     - キャラクター評価の詳細（強み・弱み・おすすめ編成）
     - 武器/アイテムの入手方法・性能数値
     - ステージ攻略のコツ・敵パターン
     - 隠し要素・小ネタ

2. 最新情報の反映
   - アップデート履歴は最新のものが一番上、古いものが一番下
   - 最新アップデートの内容がページ全体に正しく反映されているか
     （例：キャラ調整があったなら、該当キャラの評価・ランキングも更新済みか）
   - 最強ランキング・Tier表が最新環境を反映しているか
   - 古い情報がそのまま残っていないか

3. アップデート履歴セクション
   - 過去のアップデート内容は正確にそのまま記載すること
   - 日付・バージョン番号を必ず含めること
   - 新しい順（降順）で並べること
```

### A-2. アイコン・画像ルール

```
【アイコン配置ルール】
- 武器、アイテム、ステージ、キャラクターには必ず1つアイコンを設置すること
- アイコンなしのテキストのみの一覧は不可

【アイコン仕様】
- フォーマット：WebP形式（必須）
- サイズ：統一サイズ（128x128 または 64x64）
- 輪郭：くっきり太線（2-3px）
- 角丸：8px統一
- 透かし：右下に「GF」ロゴを半透明で配置
- 背景：透過 or ゲームテーマカラーグラデーション

【画像の量】
- なるべく多くの画像を使い、華やかなページにすること
- 以下の箇所には必ず画像を入れること：
  - ページトップのアイキャッチ画像
  - 各セクションの見出し付近
  - Tier表・ランキング表のキャラ/武器アイコン
  - 攻略手順の各ステップ
  - アップデート情報のハイライト

【表のレイアウト】
- アイテム名・武器名・ステージ名を表に入れる際：
  - アイコン用の列幅を十分に確保すること（最低64px幅）
  - 名称列は十分な幅を取り、名詞の途中で改行が入らないようにすること
  - 特に日本語の武器名・アイテム名は折り返さないよう min-width を設定
  - 無駄な改行を排除し、1行で収まるレイアウトにすること
  - 推奨：white-space: nowrap; をアイテム名セルに適用
```

### A-3. ナビゲーション・リンクルール

```
【内部リンク】
- ページ内に登場する以下の名詞には全てGamers-For内部リンクを貼ること：
  - キャラクター名
  - 武器名
  - アイテム名
  - ステージ名
  - 技名・スキル名
  - 素材名
- リンク先：gamers-for.github.io 内の該当ページ（存在しない場合は空ページへのリンク）
- リンクの視覚的区別：色付き + ホバーで下線表示

【目次・ジャンプナビゲーション】
- 項目数が多いカテゴリ（武器一覧、アイテム一覧等）では：
  - ページ上部にカテゴリ別目次を設置
  - 例：シューター | チャージャー | スピナー | ローラー | ...
  - 各カテゴリ名をクリックすると該当セクションにスムーズスクロール
  - 目次はスティッキー表示 or フローティングボタンで常時アクセス可能に

【パンくずリスト】
- 全ページにパンくずリストを設置
- 例：ホーム > スプラトゥーン3 > 武器一覧 > スプラシューター
```

### A-4. コメント欄

```
【コメント欄の要件】
- 全ページの最下部にコメント欄を設置
- 誰でも気軽にコメントできる仕様（アカウント不要）
- 実装方法の候補：
  - GitHub Issues連携（GitHub Pages互換）
  - utterances（GitHub Issues ベース、無料）
  - giscus（GitHub Discussions ベース、無料）
- コメント欄の上に「コメントを投稿する」ボタンを目立つように配置
- 投稿ガイドライン（簡潔に）を表示
```

## B. 制作後の品質チェックプロンプト

### B-1. 情報量・網羅性チェック

```
【チェック指示】
作成したGamers-ForのHTMLページについて、以下を徹底的に確認せよ：

1. 情報量の比較
   - 同一ゲーム・同一トピックの主要攻略サイトと比べて、
     Gamers-Forの情報量は同等以上か？
   - 不足している項目・セクションはないか？
   - 説明文が薄い箇所をリストアップせよ

2. 最新性の確認
   - アップデート履歴の最新エントリは何か？
   - その最新アップデートの内容がページ本文に反映されているか？
   - ランキング・Tier表は最新環境を反映しているか？
   - 過去のバランス調整が反映されずに古い評価が残っていないか？

3. 正確性の確認
   - 数値データ（攻撃力、HP、倍率等）に誤りがないか
   - キャラクター名・武器名のスペルミスがないか
   - リンク先が正しいか（404チェック）
```

### B-2. デザイン・レイアウトチェック

```
【チェック指示】
作成したGamers-ForのHTMLについて、以下を確認せよ：

1. 見やすさ・魅力
   - 主要攻略サイトと比較して見やすいか？
   - 色使い・フォント・余白は適切か？
   - 華やかさは十分か？画像は豊富に使われているか？

2. アイコンの充実度
   - 全ての武器・アイテム・キャラにアイコンが付いているか？
   - アイコンなしで名前だけが並んでいる箇所はないか？
   - アイコンサイズは統一されているか？

3. 表のレイアウト
   - 名詞（武器名・アイテム名等）が途中で改行されていないか？
   - アイコンと名前の間に十分なスペースがあるか？
   - 表が横に見切れていないか？（特にスマホ表示）

4. 構成・ナビゲーション
   - 情報が論理的に整理されているか？
   - 目次からの遷移はスムーズか？
   - パンくずリストは正しく機能しているか？
```

### B-3. スマホ対応（レスポンシブ）チェック

```
【チェック指示】
以下のデバイス幅で全ページの表示を徹底的に確認せよ：

1. ブレークポイント別チェック
   - 320px（iPhone SE / 小型端末）
   - 375px（iPhone 12/13/14）
   - 390px（iPhone 14 Pro）
   - 414px（iPhone 14 Plus）
   - 768px（iPad 縦向き）
   - 1024px（iPad 横向き）
   - 1280px（PC標準）
   - 1920px（PC大画面）

2. スマホ表示の必須チェック項目
   □ ハンバーガーメニューが正常に動作するか
   □ 横スクロールが発生していないか（body overflow-x確認）
   □ テキストが小さすぎないか（最低14px）
   □ タップターゲットが十分な大きさか（最低44x44px）
   □ 画像がはみ出していないか（max-width: 100%）
   □ 表が横スクロール対応になっているか（overflow-x: auto）
   □ 表の中の文字が潰れていないか
   □ 目次のタップが正確にスクロールするか
   □ フローティングボタンが他の要素と被っていないか
   □ コメント欄がスマホでも使いやすいか
   □ アイコンが小さすぎず認識可能か
   □ ページ読み込み速度（3G回線想定で3秒以内）

3. 横向き表示
   □ 画面回転時にレイアウトが崩れないか
   □ 表やTier表が横向きで見やすくなるか

4. ダークモード対応
   □ prefers-color-schemeに対応しているか
   □ ダークモード時にテキストが読みにくくなっていないか
   □ アイコンの透過背景がダークモードで浮いていないか
```

### B-4. SEO・パフォーマンスチェック

```
【チェック指示】

1. メタデータ
   □ <title>タグは適切か（ゲーム名 + トピック + Gamers-For）
   □ meta description は150文字以内で内容を要約しているか
   □ OGP（og:title, og:description, og:image）が設定されているか
   □ canonical URLが正しいか
   □ 構造化データ（JSON-LD）が入っているか

2. サイトマップ・インデックス
   □ sitemap.xmlに当該ページが含まれているか
   □ robots.txtでブロックされていないか
   □ noindex/nofollowが誤って設定されていないか

3. パフォーマンス
   □ 画像は全てWebP形式か
   □ 画像の遅延読み込み（loading="lazy"）が設定されているか
   □ CSSはインライン or 1ファイルに統合されているか
   □ 不要なJavaScriptが読み込まれていないか
   □ Lighthouse スコア（Performance 90以上、Accessibility 90以上）
   □ Core Web Vitals（LCP 2.5秒以内、CLS 0.1以下、INP 200ms以内）

4. アクセシビリティ
   □ 画像にalt属性が設定されているか
   □ 見出しの階層が正しいか（h1→h2→h3の順）
   □ リンクテキストが「こちら」等ではなく具体的か
   □ 色のコントラスト比が4.5:1以上か
   □ キーボード操作で全要素にアクセス可能か
```

### B-5. セキュリティ・法的チェック

```
【チェック指示】

1. 著作権対策
   □ 他サイトからのコピペ文章が含まれていないか
   □ 画像はGamers-Forオリジナル or 加工済み + GF透かし入りか
   □ 他サイトへの参照・引用元表記が含まれていないか（含めてはいけない）
   □ 問い合わせフォーム / 削除依頼ページが設置されているか
   □ DMCA対応用の自動削除スクリプトが準備されているか

2. プライバシー・法的要件
   □ プライバシーポリシーページが存在するか
   □ Cookie使用時の同意バナーが表示されるか（広告使用時）
   □ Google Analytics使用時のプライバシーポリシー記載
   □ 特定商取引法の表記（収益化時）

3. XSS・セキュリティ
   □ コメント欄にXSS対策が施されているか
   □ 外部スクリプトはCDN経由で integrity 属性付きか
   □ Content-Security-Policy ヘッダーの設定
```

### B-6. ユーザーエクスペリエンスチェック

```
【チェック指示】

1. ファーストビュー
   □ ページを開いた瞬間に何のゲームの何の情報かわかるか
   □ 最も重要な情報（ランキング/攻略等）がスクロールなしで見えるか
   □ 広告でコンテンツが隠れていないか

2. ナビゲーション
   □ 3クリック以内で目的のページに到達できるか
   □ グローバルナビに主要ゲームへの導線があるか
   □ 関連ページへの導線（「おすすめ記事」「関連攻略」）があるか
   □ 検索機能が動作するか

3. 読みやすさ
   □ 段落が長すぎないか（3-4文ごとに区切り）
   □ 重要な情報がハイライトされているか
   □ 初心者でも理解できる言葉遣いか
   □ 専門用語には補足説明があるか

4. 回遊率向上
   □ 記事末尾に関連記事リンクがあるか
   □ サイドバーに人気記事ランキングがあるか
   □ 「次に読むべき記事」の導線があるか
   □ カテゴリ/タグでの絞り込みが可能か

5. 更新感の演出
   □ 最終更新日時が目立つ位置に表示されているか
   □ 「本日の閲覧数」等のリアルタイム指標があるか
   □ 新着記事・更新記事バッジが付いているか
   □ 対応ゲーム数が表示されているか
```

### B-7. 競合比較チェック（最終確認）

```
【チェック指示】
同一ゲーム・同一トピックの主要攻略サイトのHTMLを取得し、
Gamers-Forの作成したHTMLと以下の観点で比較せよ：

□ 情報量：Gamers-Forの方が多いか、同等か、少ないか
□ 見やすさ：レイアウト・デザインはGamers-Forの方が優れているか
□ 画像・アイコンの量：Gamers-Forの方が豊富か
□ 表のレイアウト：名前の改行、アイコンの有無
□ ナビゲーション：目次・リンクの使いやすさ
□ スマホ表示：どちらが見やすいか
□ 広告の少なさ：Gamers-Forの方がコンテンツに集中できるか
□ 更新日時：Gamers-Forの情報は最新か

比較結果を表形式で出力し、Gamers-Forが劣っている項目を
改善タスクとしてリストアップせよ。
```

## C. 自動実行用ワンショットプロンプト

以下は、Claude Code等に一括で投げるための統合プロンプト：

```
あなたはGamers-For攻略サイトの品質管理AIです。
指定されたHTMLページに対して以下の全チェックを実施し、
結果をJSON形式で出力してください。

チェック対象URL: {TARGET_URL}
ゲームタイトル: {GAME_TITLE}

【チェック項目】
1. 情報の網羅性（全ソースの和集合になっているか）
2. 最新情報の反映（アップデート履歴の最新エントリ確認）
3. アイコン完備（全アイテム・武器・キャラにアイコンがあるか）
4. 画像フォーマット（全WebP + GF透かし）
5. 表レイアウト（名詞の改行チェック）
6. 内部リンク（全名詞にリンク貼付）
7. 目次・ジャンプナビ（大量項目ページ）
8. コメント欄（最下部に設置）
9. レスポンシブ（320px/375px/768px/1280px）
10. SEOメタデータ（title/description/OGP/構造化データ）
11. パフォーマンス（画像lazy load/CSS統合/Lighthouseスコア）
12. アクセシビリティ（alt属性/見出し階層/コントラスト比）
13. セキュリティ（XSS対策/CSP設定）
14. UX（3クリックルール/ファーストビュー/回遊導線）
15. 競合比較（情報量/デザイン/画像量で優位か）

【出力形式】
{
  "page": "{TARGET_URL}",
  "game": "{GAME_TITLE}",
  "overall_score": 85,
  "checks": [
    {
      "category": "情報網羅性",
      "status": "PASS/WARN/FAIL",
      "score": 90,
      "issues": ["〇〇セクションの情報が不足"],
      "fix_actions": ["△△の情報を追加"]
    },
    ...
  ],
  "priority_fixes": [
    "最優先で修正すべき項目リスト"
  ]
}
```

## D. ページ制作前の事前準備チェックリスト

```
□ URL slug が確定しているか（後から変更不可）
□ 対象ゲームの最新バージョン / パッチノートを確認したか
□ 必要なアイコン素材が全て準備できているか
□ WebP変換 + GF透かし挿入のパイプラインが動作するか
□ テンプレートHTMLが最新版か
□ sitemap.xml への追加スクリプトが準備されているか
□ GitHub Pages へのデプロイパイプラインが動作するか
```
