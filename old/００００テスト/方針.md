# 全自動ゲーム攻略サイト量産 作戦書（新方式）

**最終更新**: 2026-02-12
**ステータス**: 設計フェーズ
**方針**: 既存の`００テンプレート/`パイプラインとは**別の新しいアプローチ**

---

## コンセプト

**Game8・GameWith・アルテマの3大攻略サイトから同一ゲームの情報を丸ごと取得し、
3サイトを丸パクリしたシンプルなレイアウトを作り、
3サイト+SNSの情報を統合した読みやすい記事を自動生成する。**

Hugo + GitHub Pages でデプロイ。

---

## フェーズ1: 3サイトの素材を全部取得する

### やること

Game8・GameWith・アルテマの**同一タイトルのゲーム**について、以下を全部取得する:

| 取得対象 | 内容 |
|---------|------|
| **HTML** | 各ページの完全なHTMLソース |
| **CSS** | スタイルシート全て（配色、フォント、レイアウト） |
| **アイコン** | キャラアイコン、武器アイコン、属性アイコン、ティアバッジ等 |
| **画像** | バナー、ヘッダー画像、背景画像、その他全画像 |

### 取得対象ページ

3サイトそれぞれで以下のページを取得:

- ゲームTOPページ
- 最強ランキング（ティアリスト）
- キャラ一覧
- キャラ個別ページ（全キャラ分）
- 武器一覧・武器個別
- 初心者ガイド
- リセマラランキング
- イベント情報
- その他主要記事

### 出力

```
raw_data/<game_slug>/
├── game8/
│   ├── html/            # 全ページのHTMLファイル
│   ├── css/             # CSSファイル全て
│   ├── icons/           # アイコン画像全て
│   └── images/          # その他画像全て
├── gamewith/
│   ├── html/
│   ├── css/
│   ├── icons/
│   └── images/
└── altema/
    ├── html/
    ├── css/
    ├── icons/
    └── images/
```

### スクレイピング方法

- **Playwright**（ヘッドレスブラウザ）でJSレンダリング後のHTMLを取得
- ページ内のCSS（`<link>`, `<style>`）を全て保存
- ページ内の画像（`<img>`, `background-image`）を全てダウンロード
- リクエスト間隔: 2秒以上
- 一度取得したらキャッシュして再取得しない

## フェーズ2: 3サイトの項目を全て網羅した骨組み・地盤を作る

### やること

フェーズ1で取得した3サイトのHTML/CSSを元に、**3サイトが持っている項目・セクション・データ構造を全て網羅した、骨組みと地盤となるHTML**を作成する。
**省略するのは解説文（詳しい説明テキスト）だけ。項目・見出し・テーブル・アイコン枠・セクション構造は全て入れる。**

### 骨組みに含めるもの vs 省略するもの

| 含めるもの（骨組み・地盤） | 省略するもの（フェーズ3で埋める） |
|--------------------------|-------------------------------|
| 全ページの見出し・セクション構造 | 各項目の詳しい解説文 |
| ティアリストの全キャラ/武器名+アイコン枠 | 各キャラ/武器の評価コメント |
| ステータステーブルの枠（行・列・ヘッダー） | テーブル内の評価文・おすすめ理由 |
| ナビゲーション・サイドバー・パンくず | - |
| カテゴリ分け・タグ分け | - |
| アイコン表示枠（`<img>`タグ） | - |
| 強い点/弱い点の枠 | 具体的な強い点/弱い点の文章 |
| おすすめ編成の枠 | 編成の解説文 |
| SNSの評判セクションの枠 | 実際のSNS引用 |
| 関連記事リンク | - |
| メタタグ（title, description枠） | description本文 |

### 具体的な作業

1. **3サイトの項目を全て洗い出す**
   - Game8にあってGameWithにない項目 → 入れる
   - GameWithにあってアルテマにない項目 → 入れる
   - アルテマにあってGame8にない項目 → 入れる
   - **3サイトの和集合（全項目の網羅）**

2. **3サイトのHTML構造を比較する**
   - 各ページの見出し構造、セクション分け、カード配置、テーブル構造
   - ナビ、サイドバー、フッターの配置

3. **3サイトのCSSを比較する**
   - 配色（背景色、テキスト色、アクセント色、ティア色）
   - フォントサイズ、行間、余白
   - カード、テーブル、グリッドのスタイル
   - ブレークポイント（モバイル対応）

4. **3サイトの「いいとこ取り」で網羅的なHTML/CSSを作る**
   - ヘッダー/ナビ: 全カテゴリへのリンク
   - ティアリスト: 全ティアの全キャラ/武器をアイコングリッドで配置
   - キャラ個別: ステータス表、評価欄、強い点/弱い点、おすすめ編成、SNS枠（全セクション構造）
   - 一覧ページ: 全項目をテーブル or カードグリッドで配置
   - 初心者ガイド: 全ステップの見出しと枠
   - リセマラ: 全ランク・全キャラの枠
   - **解説文だけが空欄**（枠・構造・項目名・アイコンは全て入っている）

5. **Hugoテンプレートとして組み立てる**

### 出力

```
gamers-for/
├── layouts/
│   ├── _default/baseof.html      # 共通ベース（3サイト融合、全セクション網羅）
│   ├── _default/single.html      # 個別ページ（全項目の枠）
│   ├── _default/list.html        # 一覧ページ（全カテゴリ網羅）
│   ├── partials/header.html      # ヘッダー（全カテゴリナビ）
│   ├── partials/sidebar.html     # サイドバー（人気記事、カテゴリ、最新情報）
│   ├── partials/footer.html      # フッター
│   └── shortcodes/               # ショートコード
├── static/css/style.css          # 3サイト融合CSS
└── content/games/<game_slug>/
    ├── _index.md                 # ゲームTOP（全セクション枠あり、解説文だけ空）
    ├── tier-list.md              # ティアリスト（全キャラ配置済み、評価文だけ空）
    ├── characters/_index.md      # キャラ一覧（全キャラ行あり）
    ├── characters/<name>.md      # キャラ個別（全項目の枠あり、解説文だけ空）
    ├── weapons/_index.md         # 武器一覧（全武器行あり）
    ├── weapons/<name>.md         # 武器個別（全項目の枠あり）
    ├── beginner.md               # 初心者ガイド（全ステップ枠あり）
    ├── reroll.md                 # リセマラ（全ランク・全キャラ枠あり）
    └── events.md                 # イベント（セクション枠あり）
```

### フェーズ2完了時のイメージ

```
ブラウザで見ると:
- 全ページが存在する
- 全セクション・全見出しが見える
- 全キャラ/武器がアイコン枠付きで並んでいる
- テーブルの行・列が全て存在する
- ただし、解説文のところだけが空白 or プレースホルダー
  「（評価コメントはフェーズ3で記入）」

→ 骨組みと地盤が完成している状態
→ フェーズ3では文章を流し込むだけ
```

## フェーズ3: 3サイト+SNS情報をまとめた詳細記事を書く

### やること

フェーズ2で作った枠の中に、**各項目ごとに3サイトの情報とTwitterなどの情報をまとめて要約した読みやすい文章を書いていく**。アイコンも3サイトと同じように入れる。

### 具体的な作業

1. **各項目ごとに3サイトの情報を突き合わせる**
   - Game8の評価・説明
   - GameWithの評価・説明
   - アルテマの評価・説明
   - 3サイトで共通する情報 → 確定情報
   - 3サイトで異なる情報 → 複数の見解として紹介

2. **Twitter/SNSの情報も加える**
   - ゲーム名+キャラ名 で検索
   - ユーザーの生の声・使用感
   - 「SNSでの評判」セクションとして追加

3. **要約した読みやすい文章を生成する**
   - 3サイトの情報を統合して独自の文章にする
   - 箇条書き・テーブルを活用してスキャンしやすく
   - 1文80文字以内、簡潔に
   - 「複数の攻略サイトの評価を総合すると〜」のような形

4. **アイコンを3サイトと同じように配置する**
   - キャラ名の横にキャラアイコン
   - 武器名の横に武器アイコン
   - ティアリストでアイコングリッド表示
   - 属性・タイプのアイコンも表示

### 記事構成の例（キャラ個別ページ）

## Hugo + GitHub Pages（既存設定を流用）

### hugo.toml

```toml
baseURL = 'https://username.github.io/gamers-for/'
languageCode = 'ja'
title = 'Gamers-For | ゲーム攻略'
defaultContentLanguage = 'ja'
enableRobotsTXT = true
enableEmoji = false

[params]
  description = "広告最小限の見やすいゲーム攻略サイト"

[markup]
  [markup.goldmark]
    [markup.goldmark.renderer]
      unsafe = true
  [markup.tableOfContents]
    endLevel = 3
    startLevel = 2

[taxonomies]
  category = "categories"
  tag = "tags"

[outputs]
  home = ["HTML", "RSS", "JSON"]
  section = ["HTML", "RSS"]

[sitemap]
  changefreq = "daily"
  priority = 0.5
```

### GitHub Actions（`.github/workflows/hugo.yml`）

```yaml
name: Deploy Hugo site to GitHub Pages
on:
  push:
    branches: ["main"]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    env:
      HUGO_VERSION: 0.139.0
    steps:
      - name: Install Hugo CLI
        run: |
          wget -O ${{ runner.temp }}/hugo.deb \
            https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
          && sudo dpkg -i ${{ runner.temp }}/hugo.deb
      - uses: actions/checkout@v4
        with: { submodules: recursive, fetch-depth: 0 }
      - uses: actions/configure-pages@v5
        id: pages
      - name: Build with Hugo
        env:
          HUGO_ENVIRONMENT: production
          TZ: Asia/Tokyo
        run: hugo --gc --minify --baseURL "${{ steps.pages.outputs.base_url }}/"
      - uses: actions/upload-pages-artifact@v3
        with: { path: ./public }

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - uses: actions/deploy-pages@v4
        id: deployment
```

### デプロイの流れ

```
ローカルでフェーズ1→2→3を実行
  ↓
git add . && git commit -m "Add <game> pages" && git push origin main
  ↓
GitHub Actions が自動起動
  ↓
Hugo build (--gc --minify)
  ↓
GitHub Pages にデプロイ
  ↓
https://username.github.io/gamers-for/ で公開
```
